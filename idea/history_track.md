# History Track: Regulatory Intelligence Timeline

## 1. Visual Style & UI/UX Principles

The "History Track" page should follow the **Simple Scientific/Government** aesthetic: professional, high-density but readable, and focused on data integrity.

### A. Layout Architecture
- **Sidebar Timeline (Left):** A vertical "Progress Line" with nodes for each version.
    - **Active Node:** Highlighted in "FDA Blue" (`#1e40af`).
    - **Quick Stats:** Each node shows the version number and a small badge for the number of sections changed (if already analyzed).
- **Main Content Area (Right):**
    - **Header:** Sticky header showing the "Current Version" vs "Previous Version" being compared, with a "Print Report" button.
    - **The "Change Summary" Card:** Top-level executive summary generated by AI. Uses a soft yellow background (`#fef3c7`) to highlight its importance.
    - **The Diff Feed:** A scrollable list of sections where changes were detected. 
        - Clean, side-by-side or "interleaved" text blocks.
        - **Additions:** Subtle green background (`#ecfdf5`).
        - **Deletions:** Subtle red background (`#fef2f2`).

### B. Highlighting Key Information
- **Safety Signal Badges:** Use icons/colors for change types:
    - 🔴 **New Boxed Warning / Contraindication** (High Priority)
    - 🟡 **Update to Precautions / Adverse Reactions** (Medium Priority)
    - 🔵 **Dosage / Administrative Update** (Information)
- **Visual Contrast:** Use bold, sans-serif fonts for section titles (e.g., "5.1 DRUG INTERACTIONS") and a clean monospace font for the specific text fragments being compared.

## 2. AI Comparison Engine

### A. The Prompt Strategy
Since changes can occur anywhere in the document, the prompt instructs the LLM to scan the entire label while focusing on **Regulatory and Clinical Significance**.

**System Prompt:**
> You are an expert Regulatory Affairs Specialist and Clinical Pharmacist. Your task is to perform a high-fidelity comparison between two versions of a drug's Structured Product Labeling (SPL).

**User Prompt:**
> Compare the following two versions of the drug label for [DRUG NAME].
> 
> **Version A (Old):** [TEXT_A]
> **Version B (New):** [TEXT_B]
> 
> **Instructions:**
> 1. Scan the entire content of both versions.
> 2. Identify every clinical, safety, or regulatory change. Ignore minor formatting, white-space, or punctuation changes that do not alter medical meaning.
> 3. For each change, categorize it by its clinical impact (Safety, Efficacy, Administration, or General Metadata).
> 4. Determine if the change introduces a higher level of risk (e.g., a new warning).
> 
> **Output Format (JSON ONLY):**
> {
>   "executive_summary": "A 2-3 sentence overview of the most significant changes in this version update.",
>   "changes": [
>     {
>       "section_title": "e.g., WARNINGS AND PRECAUTIONS",
>       "change_type": "Addition | Deletion | Modification",
>       "impact_category": "Safety | Efficacy | Administration | Other",
>       "clinical_significance": "Explain WHY this change matters for a doctor or patient.",
>       "risk_level": "High | Medium | Low"
>     }
>   ],
>   "regulatory_notable": true/false (true if Boxed Warnings, Contraindications, or Indications were changed)
> }

## 3. Workflow & Data Persistence

### A. The "Analyze" Workflow
1. **User Request:** User clicks "Analyze Changes" on a specific version node in the timeline.
2. **Preprocessing:** The backend fetches the XML for the current and previous versions, strips redundant HTML/XML tags, and prepares a clean text diff.
3. **AI Execution:** The prompt is sent to the configured LLM (Gemini or OpenAI).
4. **Parsing & Storage:**
    - The JSON response is validated.
    - Data is stored in `labeling.history_analysis`.
5. **UI Update:** The UI immediately transitions from a "Loading" state to displaying the structured summary and badges.

### B. Database Schema (Refined)
```sql
CREATE TABLE labeling.history_analysis (
    id SERIAL PRIMARY KEY,
    set_id TEXT NOT NULL,
    current_spl_id TEXT NOT NULL UNIQUE,
    previous_spl_id TEXT,
    executive_summary TEXT, -- Extracted from JSON for quick indexing
    is_regulatory_notable BOOLEAN DEFAULT FALSE,
    analysis_json JSONB, -- The full structured report
    raw_prompt_version TEXT, -- Tracking which prompt version was used
    last_analyzed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. Interaction Design
- **Hover to Highlight:** Hovering over an AI-detected change in the summary should auto-scroll the "Diff Feed" to the corresponding section of the label.
- **Manual Override/Refresh:** If the user feels the AI missed a nuance, the "Re-run Analysis" button will clear the cache and re-trigger the prompt with a "High Temperature" setting or an updated model.
